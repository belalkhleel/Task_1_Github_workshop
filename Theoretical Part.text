
a. git status:The git status command displays the state of the working directory and the staging area. It lets you see which changes have been staged,
   which haven’t, and which files aren’t being tracked by Git. 

b. git init:The git init command creates a new Git repository.
    It can be used to convert an existing, unversioned project to a Git repository or initialize a new, empty repository.

c. git commit:The git commit command captures a snapshot of the project's currently staged changes.

d. git push:The git push command is used to upload local repository content to a remote repository.
